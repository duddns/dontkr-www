LLM 개념 정리

2025년 11월 11일

1. LLM이란 무엇인가?#

정의#

Large Language Model = 대규모 언어 모델

  • 엄청난 양의 텍스트로 학습한
  • 파라미터가 수백억~조 개인
  • 언어를 이해하고 생성하는 AI 모델

본질#

“다음 단어 예측"을 극한까지 잘하는 모델

입력: "오늘 날씨가 정말"
LLM: "좋네요" (가장 자연스러운 다음 단어 선택)

→ 이 단순한 원리로 대화, 요약, 번역, 코딩까지 가능

2. LLM의 3가지 핵심 특징#

Large (대규모)#

파라미터 = 모델의 학습 가능한 내부 값

  • GPT-3: 1,750억 개
  • GPT-4: 추정 1조+ 개
  • Claude 3.5: 비공개 (아마 수천억~조)

많을수록:

  • 복잡한 패턴 학습
  • 미묘한 뉘앙스 이해
  • 추론 능력 향상

Language (언어)#

텍스트만 다룸

  • 학습 데이터: 텍스트
  • 입력: 텍스트
  • 출력: 텍스트
  • 이미지/음성 모델과 구분되는 핵심

Model (모델)#

머신러닝으로 학습된 결과물

  • 사람이 규칙을 짜지 않음
  • 데이터에서 스스로 패턴 학습
  • 한번 학습하면 계속 사용

3. LLM은 어떻게 만들어지나?#

1단계: Pre-training (사전 학습)#

데이터#

  • 인터넷 전체 텍스트 (수조 단어)
    • 위키백과
    • 뉴스 기사
    • 책
    • 코드
    • 웹페이지
    • 대화 등

방법#

  • “다음 단어 맞추기” 반복 학습

기간#

  • 수개월 (엄청난 컴퓨팅 파워)

비용#

  • 수백억 원

결과#

  • 언어의 구조와 패턴 이해
  • 세상 지식 습득
  • 하지만 대화는 아직 서툴음

2단계: Fine-tuning (미세 조정)#

데이터#

  • 고품질 대화/지시 데이터
    • 사람이 직접 작성한 대화
    • “좋은 답변” vs “나쁜 답변” 평가

방법#

  • RLHF (인간 피드백 강화학습)

기간#

  • 수주~수개월

결과#

  • 자연스러운 대화 가능
  • 유용하고 안전한 답변
  • 사용자 의도 이해

4. LLM의 작동 원리 (간단히)#

핵심: Transformer 구조#

Attention 메커니즘#

"그는 학교에 갔다. 그는 친구를 만났다."
        ↑                    ↑
        └────────관계 파악────┘
  • 문장의 모든 단어 간 관계 분석
  • 문맥을 이해하는 핵심 기술

층층이 쌓인 구조#

입력 텍스트
    ↓
[Transformer Layer 1] ← 간단한 패턴
[Transformer Layer 2] ← 조금 복잡한 패턴
[Transformer Layer 3] ← 더 복잡한 패턴
    ...
[마지막 Transformer Layer] ← 추상적 의미
    ↓
출력 텍스트

토큰 단위 처리#

"안녕하세요" → ["안녕", "하세요"] (2토큰)
→ 각 토큰마다 확률 계산
→ 가장 적절한 다음 토큰 선택

5. LLM이 잘하는 것#

자연스러운 대화#

  • 문맥 유지
  • 의도 파악
  • 자연스러운 응답

텍스트 생성/변환#

  • 요약
  • 번역
  • 다시 쓰기
  • 형식 변환

지식 활용#

  • 질문 답변
  • 설명
  • 조언

추론#

  • 논리적 사고
  • 단계별 분석
  • 문제 해결

창작#

  • 글쓰기
  • 아이디어 발산
  • 스토리텔링

코딩#

  • 코드 생성
  • 버그 수정
  • 설명

6. LLM의 한계#

할루시네이션 (가장 큰 문제)#

사실이 아닌 내용을 그럴듯하게 생성

질문: "2025년 노벨물리학상 수상자는?"
LLM: "김철수 박사입니다" (지어낸 답)

→ 자신 없어도 확신하는 척
→ 중요한 사실은 반드시 검증 필요

지식 컷오프#

학습 시점 이후 정보 모름

  • Claude: 2025년 1월까지만 앎
  • 최신 뉴스/사건 모름
  • 검색 도구로 보완 가능

계산 약함#

수학 계산 실수 가능

질문: "123456 × 789 = ?"
LLM: 틀릴 수 있음

→ 복잡한 계산은 코드/도구 사용

일관성 문제#

  • 같은 질문에 다른 답변
  • 긴 대화에서 앞뒤 모순

편향#

  • 학습 데이터의 편향 반영
  • 문화/성별/인종 편견 가능

비용#

  • API 호출마다 과금
  • 대량 사용 시 비용 부담

7. LLM과 다른 AI의 관계#

전체 지형도#

AI
├─ 전통 머신러닝
│   └─ 추천 시스템 (넷플릭스)
├─ 딥러닝
│   ├─ CNN → 이미지 인식
│   ├─ RNN → 시계열
│   ├─ Transformer
│   │   └─ LLM ← 여기
│   ├─ Diffusion → 이미지 생성 (Stable Diffusion)
│   └─ GAN → 이미지/영상 생성
└─ 강화학습 → 게임 AI

LLM의 위치#

  • AI의 일부
  • 딥러닝의 일부
  • Transformer 기반
  • 현재 가장 주목받는 분야

Multimodal 확장#

최근 트렌드:

LLM (텍스트)
    +
Vision (이미지)
    +
Audio (음성)
    =
Multimodal AI (GPT-4V, Gemini)

→ 하지만 코어는 여전히 LLM

8. 핵심 요약#

LLM#

  • 엄청난 텍스트로 학습한, 언어를 이해하고 생성하는 대규모 AI 모델

특징#

  • Large: 수백억~조 개 파라미터
  • Language: 텍스트만 처리
  • 다음 단어 예측이 핵심 원리

잘하는 것#

  • 대화, 요약, 번역, 추론, 창작, 코딩

한계#

  • 할루시네이션, 지식 컷오프, 계산 약함, 비용

위치#

  • AI > 딥러닝 > Transformer > LLM
