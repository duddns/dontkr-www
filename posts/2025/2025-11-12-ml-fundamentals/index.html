2. 핵심 용어 완전 정리#

데이터 (Data)#

원본 정보

예시#

  • 텍스트: “안녕하세요”
  • 이미지: 고양이 사진
  • 표: 집 정보 (평수, 방개수)

역할#

학습의 재료

수치화 (Numericalization)#

데이터를 숫자로 변환하는 과정 (표현 방식의 변환)

"안녕" → 1523 (숫자 ID)
빨간 픽셀 → [255, 0, 0] (RGB 값)

목적#

기계가 읽을 수 있는 형태로 변환

Feature 추출 (Feature Extraction)#

수치화된 데이터 중 의미있는 부분을 선택하거나 새로 만드는 과정

수치화: [1523, 8842, ...]
    ↓
Feature 추출: 의미있는 표현 생성
    ↓
Feature: [[0.2, -0.5, 0.8], ...]

핵심#

  • 수치화는 표현 방식의 변환
  • Feature 추출은 의미 선택의 과정
  • 둘은 구분되지만 일부 겹침

Feature (특징)#

모델이 학습에 활용하는 의미있는 숫자 표현

전통 머신러닝 예시 (감정 분석)#

리뷰 텍스트 → Feature (사람이 설계)
- 긍정 단어 개수: 3
- 부정 단어 개수: 0
- 느낌표 개수: 2
→ Feature: [3, 0, 2]

딥러닝 예시 (감정 분석)#

리뷰 텍스트 → Feature (자동 학습)
- 입력: "이 영화 정말 최고예요!"
- Layer 1: 단어 조합 패턴
- Layer 2: 감정 강도 패턴
- Layer 3: 전체 의도 (극찬)

학습 대상 여부#

  • 전통 머신러닝: Feature 고정, Parameter만 학습
  • 딥러닝: Parameter 학습을 통해 Feature 표현도 간접 학습

핵심#

  • 모델에 “무엇을 볼 것인가” 제공
  • 딥러닝: 각 Layer 출력도 Feature

전통 머신러닝 vs 딥러닝#

  • 전통 머신러닝: 사람이 Feature 설계 (고정)
  • 딥러닝: Feature 자동 학습 (Parameter로 제어)

Parameter (파라미터)#

모델이 학습으로 조정하는 내부 숫자 값

예시#

감정점수 = (긍정단어 × a) + (부정단어 × b) + (느낌표 × c)

a = 0.5   ← Parameter
b = -0.8  ← Parameter
c = 0.3   ← Parameter

구성#

  • 가중치 (Weight)
  • 편향 (Bias)

핵심#

  • 학습으로 조정됨 (최적화 대상)
  • Feature를 “어떻게 처리할지” 결정

크기#

구분 Parameter 규모 예시
전통 머신러닝 10² ~ 10⁴ 선형 회귀, SVM
CNN (딥러닝) 10⁵ ~ 10⁷ 이미지넷 모델
Transformer 10⁸ ~ 10¹⁰ BERT, GPT-2
LLM 10¹¹ ~ 10¹³ GPT-3, GPT-4

범위#

머신러닝 전체 (딥러닝 전용 아님)

토큰 (Token)#

텍스트를 처리 가능한 작은 단위로 나눈 것

"안녕하세요" → ["안녕", "하세요"] (2토큰)
"I love you" → ["I", "love", "you"] (3토큰)

특징#

  • NLP(자연어 처리) 전용 용어
  • 이미지는 “픽셀”, 음성은 “프레임”
  • LLM에서 비용/길이 계산 단위

범위#

자연어 처리 전반 (LLM 전용 아님)

학습 (Training)#

데이터로 최적의 Parameter를 찾는 과정

  1. 랜덤 Parameter로 시작
  2. Feature × Parameter = 예측
  3. 오차 계산 (틀린 정도)
  4. 역전파로 Parameter 조정
  5. 반복 (점점 정확해짐)

결과#

최적화된 Parameter

딥러닝에서#

Parameter 학습으로 Feature 표현도 함께 최적화됨

추론 (Inference)#

학습된 모델로 예측하는 과정

입력 Feature × 학습된 Parameter = 예측 결과

예시#

  • 새 집 정보 → Feature 추출 → 모델로 계산 → 집값 예측
  • 고양이 사진 → Feature 추출 → 모델로 계산 → “고양이 95%”

모델 (Model)#

구조(Architecture) + 학습된 Parameter

모델 = 구조 + 학습된 Parameter

모델 파일 (예: PyTorch .pt, TensorFlow .pb):
- 네트워크 구조 정의
- 학습된 Parameter 값들

예시#

GPT-3 모델 = Transformer 구조 + 1,750억 개 Parameter = 700GB

사용#

새 입력 → 모델(구조+Parameter) → 예측

Feature vs Parameter 한눈에#

Feature Parameter
정의 의미있는 숫자 표현 학습 가능한 내부 값
출처 데이터에서 계산 학습으로 조정
역할 무엇을 볼 것인가 어떻게 계산할 것인가
예시 평수 30, 픽셀값 a=200, w=1.5
위치 입력/중간 표현 모델 내부
학습 대상 전통 머신러닝: ✗딥러닝: 간접 ✓ ✓ (직접 최적화)
비유 요리 재료 요리 레시피

관계#

Feature (입력) × Parameter (가중치) = 출력

3. 전통 머신러닝 vs 딥러닝 비교#

전통 머신러닝#

원본 데이터: "이 영화 정말 최고예요!"
    ↓
전처리 (토큰화, 정제)
    ↓
사람이 Feature 설계 (수동)
   "긍정 단어, 부정 단어, 느낌표 개수를 사용하자"
    ↓
Feature: [3, 0, 2] (고정)
    ↓
학습으로 Parameter 찾기 (자동)
    ↓
Parameter: a=0.5, b=-0.8, c=0.3
    ↓
모델: 감정점수 = 3×0.5 + 0×(-0.8) + 2×0.3 = 2.1 (긍정)

특징:

  • Feature 고정, Parameter만 학습
  • Feature 설계가 핵심 (Feature Engineering)
  • Parameter 수: 10² ~ 10⁴
  • 빠르고 해석 가능
  • 단순한 패턴 학습

예시 알고리즘:

  • 선형 회귀
  • 로지스틱 회귀
  • SVM
  • 의사결정나무
  • 랜덤포레스트

딥러닝#

원본 데이터: "이 영화 정말 최고예요!"
    ↓
전처리 (토큰화, 정제)
    ↓
Feature 자동 학습
   층층이 쌓인 신경망
    ↓
[Layer 1] 단어 조합 패턴
    ↓
[Layer 2] 감정 표현 패턴
    ↓
[Layer 3] 전체 의도/맥락
    ↓
Parameter 자동 학습
    ↓
모델: 10⁵ ~ 10¹³ 개 Parameter

특징:

  • 데이터 품질 관리와 전처리는 여전히 중요
  • Feature Engineering 불필요 (자동 학습)
  • Parameter 학습으로 Feature도 자동 학습
  • Parameter 수: 10⁵ ~ 10¹³
  • 복잡한 패턴 학습 가능
  • GPU 필요

예시 알고리즘:

  • CNN (이미지)
  • RNN (시계열)
  • Transformer (언어)
  • LLM (GPT, Claude)

비교표#

전통 머신러닝 딥러닝
Feature 사람이 설계 (고정) Parameter 학습으로 자동 학습
Parameter 자동 학습 자동 학습
파라미터 수 10² ~ 10⁴ 10⁵ ~ 10¹³
계산량 적음 많음 (GPU 필요)
데이터 필요량 수천~수만 수백만~수조
학습 시간 분~시간 일~월
복잡도 단순 패턴 복잡한 패턴
전처리 필요 필요 (자동≠불필요)
해석 가능성 높음 상대적으로 낮음 (XAI로 개선 중)

4. LLM 특수성#

LLM = 딥러닝의 한 종류#

AI
└─ 머신러닝
    ├─ 전통 머신러닝
    └─ 딥러닝
        └─ Transformer
            └─ LLM ← 여기

LLM 프로세스#

━━━━━━━━━━━━━━━━━━━━━━━
 1단계: Pre-training (사전학습)
━━━━━━━━━━━━━━━━━━━━━━━

인터넷 전체 텍스트 (수조 단어)
    ↓
토큰화
"안녕하세요" → ["안녕", "하세요"]
    ↓
임베딩 (Feature 추출)
[[0.2, -0.5, 0.8], [0.1, 0.3, -0.2]]
    ↓
━━━━━━━━━━━━━━━━━━━━━━━
 Transformer Layer 반복
━━━━━━━━━━━━━━━━━━━━━━━
    ↓
[Layer 1] Feature → 다음 Feature
[Layer 2] Feature → 다음 Feature
...
[마지막 Layer] Feature → 최종 표현
    ↓
"다음 토큰 확률 예측" 반복 학습
    ↓
10¹¹ ~ 10¹³ 개 Parameter 최적화
    ↓
━━━━━━━━━━━━━━━━━━━━━━━
 사전학습 모델 완성
━━━━━━━━━━━━━━━━━━━━━━━
    ↓
━━━━━━━━━━━━━━━━━━━━━━━
 2단계: Fine-tuning (미세조정)
━━━━━━━━━━━━━━━━━━━━━━━
    ↓
① Supervised Fine-tuning (SFT)
   - 고품질 대화 데이터로 학습
   - 지시 따르기 능력 향상
    ↓
② RLHF (인간 피드백 강화학습)
   - 사람이 "좋은 답변" 평가
   - 유용성, 안전성 향상
    ↓
━━━━━━━━━━━━━━━━━━━━━━━
 대화형 LLM 완성
━━━━━━━━━━━━━━━━━━━━━━━
    ↓
GPT-4, Claude 3.5 등

핵심:

  • 기본: 다음 토큰 확률 예측 학습
  • 완성: SFT + RLHF로 대화형 모델 발전

LLM 추론#

"오늘 날씨가"
    ↓
토큰화: ["오늘", "날씨", "가"]
    ↓
임베딩: Feature 추출
    ↓
수십~수백개 Layer 통과
(각 Layer마다 새로운 Feature 생성)
    ↓
다음 토큰 확률 분포 계산
"좋네요" (70%), "나쁘네요" (20%), "흐리네요" (8%), ...
    ↓
Sampling 전략에 따라 선택
(temperature, top-k, top-p 등)
    ↓
"좋네요" 선택

추론 메커니즘:

  • 확률 분포 계산 후
  • 설정된 sampling 전략에 따라 토큰 선택
  • Temperature 높음 → 창의적 (다양한 선택)
  • Temperature 낮음 → 보수적 (높은 확률만 선택)

LLM의 핵심 특징#

1. Large (대규모 Parameter)#

규모: 10¹¹ ~ 10¹³ 개

  • GPT-3: 1.75 × 10¹¹ 개 (700GB)
  • GPT-4: 추정 10¹² ~ 10¹³ 개

2. Language (언어 전용)#

  • 입력: 텍스트
  • 처리: 토큰
  • 출력: 텍스트

3. Model (학습된 Parameter)#

  • 사전학습: “다음 토큰 예측”
  • 미세조정: SFT + RLHF
  • → 대화, 요약, 번역, 코딩 모두 가능

5. 전체 요약#

머신러닝 = 데이터로 모델 만들기#

데이터 → Feature → Parameter 학습 → 모델 → 예측

핵심 개념#

수치화 vs Feature 추출:

  • 수치화: 표현 방식의 변환 (숫자로)
  • Feature 추출: 의미 선택의 과정 (중요한 것만)

Feature (특징):

  • 데이터에서 계산한 의미있는 숫자
  • “무엇을 볼 것인가”
  • 전통 머신러닝: 사람 설계 (고정)
  • 딥러닝: Parameter 학습으로 간접 학습

Parameter (파라미터):

  • 학습으로 조정하는 모델 내부 값
  • “어떻게 계산할 것인가”
  • 모델의 실체

학습:

  • Feature를 이용해 Parameter 최적화
  • 딥러닝: Parameter 변화로 Feature도 발전

추론:

  • 학습된 Parameter로 예측

모델:

  • 구조(Architecture) + 학습된 Parameter

발전 과정#

2000년대: 전통 머신러닝

  • 사람이 Feature 설계
  • Parameter 10² ~ 10⁴ 개
  • 간단한 작업

2010년대: 딥러닝

  • Feature 자동 학습
  • Parameter 10⁵ ~ 10⁷ 개
  • 이미지, 음성 인식 성공
  • GPU 필수

2020년대: LLM

  • 초대규모 Parameter (10¹¹ ~ 10¹³)
  • 언어 이해/생성
  • Pre-training + Fine-tuning
  • 범용 AI의 시작

6. 실무 관점 정리#

AI 도입 시 이해해야 할 것#

1. 어떤 AI인가?#

  • 전통 머신러닝: 정형 데이터, 예측
  • 딥러닝: 이미지, 음성, 복잡한 패턴
  • LLM: 텍스트 이해/생성

2. Feature는 무엇인가?#

  • 전통 머신러닝: 직접 설계 필요 (Feature Engineering)
  • 딥러닝/LLM: 자동 학습 (하지만 데이터 품질 관리와 전처리는 여전히 중요)

3. 리소스는?#

  • 전통 머신러닝: 적음
  • 딥러닝: GPU 필요
  • LLM: API 사용 (외부 모델) 또는 막대한 인프라

4. Parameter는?#

모델 크기 = Parameter 수
클수록: 성능↑, 비용↑, 속도↓

10² ~ 10⁴: 전통 머신러닝
10⁵ ~ 10⁷: CNN
10⁸ ~ 10¹⁰: BERT급
10¹¹ ~ 10¹³: LLM

5. 모델 이해#

모델 = 구조 + Parameter
